这是博客的第一篇文章，用来验证从本地写作到 GitHub Pages 自动部署的完整流程。

## 为什么选择 Hugo + PaperMod

- Hugo 构建速度快，内容组织清晰
- PaperMod 风格简洁，适合技术写作
- Markdown 工作流轻量，适配版本管理

## 本站工作流

1. 在 `content/posts/` 中写 Markdown
2. 本地用 `hugo server` 实时预览
3. 推送到 `main` 分支后由 GitHub Actions 自动部署
